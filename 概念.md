关键数据流

PyTorch模型 → DispatchExtractor → ComputeGraph

1. 将模型移动到torch.device("meta")（仅形状，无实际内存）

2. _ModuleTracker钩子将每个操作与其源子模块路径关联

3. _RecordingDispatch拦截每个aten操作，记录输入/输出TensorSpecs

4. 可选跳过零成本重塑操作；_resolve_source()跟随passthrough链到实际生产者

5. 按层次命名操作，例如“L3.self_attn.mm”（dispatch_extractor.py:547）

HuggingFace配置 → ConfigExtractor → ComputeGraph

1. normalize_config()将原始字典映射到ModelConfig

2. 根据model_type从_handlers注册表中选择架构处理器

3. GraphBuilder DSL构建嵌入 → N × transformer层 → 最终规范 + LM头

4. 每个层：处理器调用builder.attention() + FFN/MoE块

---

显著设计模式

- 元设备执行（dispatch_extractor.py）：允许从100B+参数模型中提取，而不需要实现权重

- Passthrough链解析（dispatch_extractor.py:531）：跳过重塑操作被移除；边跟随通过到实际生产者

- 量化感知构建器（graph_builder.py:53）：linear()与QuantConfig插入显式dequant节点为W4A8

- 注意力抽象：AttentionPattern将评分策略与投影类型解耦；attention()在MLA/GQA × 稠密/top_k/滑动窗口组合间分发
