好问题，这三个概念是 PyTorch 中定义算子的**三种不同层次**：

## 1. `nn.Module` — 模块层（高层封装）

`nn.Module` 是 PyTorch 的**网络层基类**，包含参数（weights）和 `forward()` 方法。

```python
import torch.nn as nn

# 定义一个线性层，内部自动创建 weight 和 bias 参数
linear = nn.Linear(in_features=256, out_features=512)

# 调用时执行 forward()
x = torch.randn(32, 256)
out = linear(x)  # 等价于 x @ weight.T + bias
```

常见模块：
| 模块 | 作用 |
|------|------|
| `nn.Linear` | 全连接层 `y = xW^T + b` |
| `nn.Conv2d` | 2D 卷积 |
| `nn.LayerNorm` | 层归一化 |
| `nn.ReLU` | ReLU 激活 |
| `nn.Softmax` | Softmax |

## 2. `torch.nn.functional` — 函数层（无状态）

和 `nn.Module` 做**同样的计算**，但不保存参数，需要手动传入 weight/bias。

```python
import torch.nn.functional as F

# 手动创建参数
weight = torch.randn(512, 256)
bias = torch.randn(512)

x = torch.randn(32, 256)
out = F.linear(x, weight, bias)  # 和 nn.Linear 计算完全一样
```

## 3. `aten::` — 底层算子（ATen 库）

ATen (A Tensor Library) 是 PyTorch 的**C++ 底层算子库**。所有高层调用最终都会落到 aten 算子。

```
nn.Linear(x)
    ↓ 调用
F.linear(x, weight, bias)
    ↓ 调用
torch.ops.aten.linear.default(x, weight, bias)
    ↓ 调用
C++ aten::linear()  ← 最底层
```

## 三者关系

```
用户代码
    │
    ├── nn.Linear (有参数，推荐用于定义网络结构)
    │       ↓ forward() 内部调用
    │   F.linear(x, self.weight, self.bias)
    │       ↓ 最终调用
    │   aten::linear
    │
    └── F.linear (无参数，灵活但需手动管理权重)
            ↓ 最终调用
        aten::linear
```

## 在 TorchGraphExtractor 中的映射

`TorchGraphExtractor` 用 `torch.fx` 追踪模型时，会遇到两种调用形式，需要分别映射到统一的 `aten::` 名称：

```python
# 代码中可能出现的两种写法：

# 写法1: 函数式调用
out = torch.nn.functional.relu(x)        # → 映射为 "aten::relu"
out = torch.nn.functional.linear(x, w)   # → 映射为 "aten::linear"

# 写法2: 模块式调用
model = nn.Sequential(nn.ReLU(), nn.Linear(256, 512))
out = model(x)  # fx 会识别为 "call_module"
                # → nn.ReLU 映射为 "aten::relu"
                # → nn.Linear 映射为 "aten::linear"
```

两种写法在 FX 图中的表现形式不同：

| 写法 | FX node.op | FX node.target | 映射方式 |
|------|-----------|----------------|---------|
| `F.relu(x)` | `call_function` | `torch.nn.functional.relu` | 查 `_FUNC_MAP` |
| `nn.ReLU()(x)` | `call_module` | `submod_0` (模块名) | 查 `_MODULE_MAP` (通过 `type(submod)`) |

## 实际例子

```python
import torch
import torch.nn as nn
import torch.nn.functional as F

class MyModel(nn.Module):
    def __init__(self):
        super().__init__()
        self.fc1 = nn.Linear(256, 512)   # nn.Module 形式
        self.fc2 = nn.Linear(512, 10)
    
    def forward(self, x):
        x = self.fc1(x)                   # call_module → aten::linear
        x = F.relu(x)                     # call_function → aten::relu
        x = F.softmax(x, dim=-1)          # call_function → aten::softmax
        x = self.fc2(x)                   # call_module → aten::linear
        return x

# TorchGraphExtractor 追踪后得到:
# Node 1: "aten::linear"  (来自 self.fc1)
# Node 2: "aten::relu"    (来自 F.relu)
# Node 3: "aten::softmax" (来自 F.softmax)
# Node 4: "aten::linear"  (来自 self.fc2)
```

**总结**：`nn.Module` 和 `F.xxx` 是用户层的两种写法，`aten::` 是统一的底层算子。`TorchGraphExtractor` 的任务就是把这两种写法都归一化到 `aten::` 级别，方便后续做性能分析。
